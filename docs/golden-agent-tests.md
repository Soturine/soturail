# Golden Agent Tests

Golden agent tests are snapshot-like checks for agent-facing outputs. They are not tests of a real LLM. They protect SotuRail exports, reports and context packs from accidental regressions.

## Golden Targets

Planned golden targets:

- `AGENTS.md` guidance generated by `harness init`;
- Claude, Codex, Cursor, OpenCode, Gemini and generic exports;
- host matrix JSON;
- MCP host manifest JSON;
- report and dashboard summaries;
- skill export bundles;
- tasklet definitions;
- evidence/provenance sidecars.

## What A Golden Test Should Check

Golden tests should favor semantic assertions over brittle full-file string matches:

```txt
must include: local-first boundary
must include: safe next commands
must include: read-only MCP warning
must include: evidence paths when available
must not include: secrets
must not include: destructive tool promises
must not include: unrelated project identity
must parse: JSON artifacts
must keep: schemaVersion
```

## Update Policy

Golden outputs should be updated deliberately:

```bash
soturail eval golden --update
soturail eval report
soturail release check --strict --with-evals
```

A golden change should be explained in release evidence. If the change affects host behavior, update host docs too.
