# Agent Docs Hygiene

Agent Docs Hygiene is the SotuRail area for keeping root agent instruction files short, current and useful.

The main idea:

```txt
Root agent docs should route context.
They should not become huge project wikis.
```

## Target Files

SotuRail should eventually inspect files such as:

- `CLAUDE.md`
- `AGENTS.md`
- `GEMINI.md`
- `.cursor/rules/*`
- `agent_docs/*`
- `.soturail/context/*`

## v0.5.0 Seed Commands

```bash
soturail agents lint
soturail agents split-context --dry-run
soturail agents explain --agent all
```

These commands report and suggest. They do not rewrite agent docs.

## v0.6.0 Runtime Adapter Checks

v0.6.0 adds host-aware inspection without making SotuRail the agent runtime:

```bash
soturail agents capabilities
soturail agents status
soturail agents doctor --verbose
soturail agents install --agent claude --dry-run
```

Use them to confirm which host surfaces are real, experimental, prompt-only or planned before adding generated files to a project.

## Why Root Agent Docs Stay Short

Root docs are loaded early and often. They should identify the project, safe commands and where to find richer context. Long root docs increase context cost, go stale quickly and can bury the task-specific evidence an agent actually needs.

Use this split:

- **Root instructions**: project identity, safety rules, build/test/release commands and links.
- **Referenced docs**: deeper workflows, architecture, policies and migration notes.
- **Context packs**: task or target-specific handoff material generated by SotuRail.
- **Skills**: reusable operating procedures loaded only when relevant.
- **MCP resources**: read-oriented local resources exposed through an explicit capability boundary.

Host-aware setup examples:

- `CLAUDE.md`: short root instructions plus references to `.soturail/context/` and optional reviewed `.claude/settings.json`.
- `AGENTS.md`: short Codex-compatible prompt-only guidance and safe command list.
- `GEMINI.md`: short Gemini-compatible guidance and context budget reminders.
- Cursor rules: small `.cursor/rules/soturail.mdc` file that points to generated context packs.
- Antigravity/OpenCode/Amp/Kiro-style hosts: prompt-only exports until stable project config surfaces are confirmed.

## What To Check

Agent docs linting should warn about:

- root instruction files that are too long;
- repeated or conflicting rules;
- stale commands;
- missing build/test/release commands;
- missing project purpose;
- missing safety rules;
- probable secrets;
- too much generic advice;
- unclear module ownership;
- dead links to docs that do not exist;
- large docs that should be referenced instead of pasted.

## Recommended Shape

A healthy setup should look like this:

```txt
CLAUDE.md or AGENTS.md
  -> short project purpose
  -> safe commands
  -> important rules
  -> links to larger docs

agent_docs/
  -> detailed workflows
  -> architecture notes
  -> testing policy
  -> release policy

.soturail/context/
  -> generated target-aware context packs
```

## Examples

`AGENTS.md`:

```md
# Agent Instructions

SotuRail is a local-first Context OS for AI coding agents.

Use safe local commands:

- npm run typecheck
- npm run build
- npx vitest run tests/v050.test.ts

For richer context, run `soturail context select --query "<task>"` or read `docs/context-intelligence.md`.
Do not expose secrets, publish packages or create GitHub releases without explicit human approval.
```

`CLAUDE.md`:

```md
# Claude Project Notes

Keep root context short. Prefer role packs:

- soturail context pack --role planner
- soturail context pack --role reviewer

MCP is read-oriented by default. Do not assume arbitrary shell execution is exposed.
```

`GEMINI.md`:

```md
# Gemini Project Notes

Use `soturail agents explain --agent gemini` for local integration guidance.
Use `soturail context budget --explain` before large tasks.
```

Cursor rules can stay as a small rule file that links to generated context:

```md
Use `.soturail/context/cursor-context.md` for generated task context.
Do not paste raw logs or secrets into rules.
```

## Relationship With Context Packs

Agent docs should point to SotuRail context packs instead of copying every detail.

Example future flow:

```bash
soturail agents lint
soturail agents split-context --dry-run
soturail agents explain --agent all
soturail context select --query "prepare npm release"
soturail context pack --role release-manager
soturail agents export --agent claude
```

## Acceptance Criteria

Agent Docs Hygiene should not be promoted until:

- lint output is actionable;
- docs are not rewritten without dry-run or backup;
- host differences are documented;
- generated suggestions stay local;
- probable secrets are warned about;
- tests cover at least `CLAUDE.md`, `AGENTS.md` and Cursor-style rules.
